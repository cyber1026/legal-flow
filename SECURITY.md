# 安全策略

Legal Flow 涉及合同文本、法律语料、用户会话和 LLM 服务密钥。请在部署前完成安全加固，不要直接使用开发默认配置暴露到公网。

## 支持版本

当前项目仍处于早期开发阶段，安全修复以 `main` 分支为准。

## 报告安全问题

请不要在公开 issue 中披露漏洞细节。可以先通过 GitHub 私有联系方式联系维护者，或提交只描述影响范围的 issue，等待维护者建立私下沟通渠道。

报告时建议包含：

- 影响模块
- 复现步骤
- 潜在影响
- 已知缓解方式

## 部署安全清单

- 设置强随机 `JWT_SECRET`，生产环境不要使用默认值。
- 不要提交 `.env`、日志、上传合同、向量库和模型权重。
- 为 PostgreSQL、Milvus、MinIO 和 embedding 服务配置网络隔离。
- 对上传文件做大小、类型、病毒扫描和内容审计。
- 为合同审查结果保留审计日志，并明确人工复核责任。
- 对外部 LLM 调用进行脱敏、限流和超时控制。
